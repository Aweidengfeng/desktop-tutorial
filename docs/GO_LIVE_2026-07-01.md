## P0-1 管理后台 CSRF 修复（已完成，记录方案）

**问题**：`adminAuth` 同时接受 `adminToken` Cookie 与 `Authorization: Bearer`。Cookie 会话此前仅依赖 `SameSite=Strict` 单层防护，缺少独立 CSRF 令牌。

**方案（双提交 CSRF Token + 统一 ******
- 登录时（`POST /api/admin/login`）除 `adminToken`（httpOnly）外，下发可被 JS 读取的 `adminCsrf` Cookie（`sameSite=strict`、生产 `secure`），并在响应体返回 `csrfToken`。
- `adminAuth` 对**基于 Cookie 的会话**的所有状态变更请求（POST/PUT/PATCH/DELETE）强制校验 `X-CSRF-Token` 头与 `adminCsrf` Cookie 一致（常量时间比较）。
- 使用 `Authorization: Bearer` 的 API / 移动端客户端予以豁免（浏览器不会自动附带 Authorization 头，无 CSRF 风险）。
- 安全方法（GET/HEAD/OPTIONS）无需令牌；存量会话在首个 GET 请求时惰性补发 `adminCsrf`，避免被动失效。
- 前端：`admin.html`（`api()` 助手 + 内联 logout / SOS resolve / 远征状态切换）与 `src/admin/api/client.ts`（axios 请求拦截器）自动回填 `X-CSRF-Token`。

**影响范围**：管理后台鉴权与所有管理端写操作；用户端 API（已是 Bearer）不受影响。
**审核影响评估**：CSRF 属服务端 Web 安全，不构成 App Store / Google Play 审核阻断项，但属上线安全基线必须项。

## P0-4 iOS / Android 原生工程重建清单

> **现状**：`ios/` 仅含 `ExportOptions.plist`，`android/` 仅含 `app/google-services.json`——**两者均非完整可构建原生工程**，需用 Capacitor 重新生成后回贴自定义配置。

### 通用前置
- [ ] Node.js 20+；`npm install`（Capacitor 依赖已固化为精确版本）。
- [ ] `npm run build` 产出 `www/`（webDir）。
- [ ] 校验 `capacitor.config.json`：`appId=app.summitlink`、`webDir=www`、`allowNavigation` 已收敛为 `summitlink.app` / `*.summitlink.app`。

### iOS 重建
- [ ] `npx cap add ios` 重新生成 `ios/App` 工程。
- [ ] 回贴 `ios/ExportOptions.plist` 与 `ios-permissions-template`（相机、定位、推送、生物识别等 `NSxxxUsageDescription`）到 `Info.plist`。
- [ ] Xcode Signing：Team ID、Bundle ID `app.summitlink`、Capabilities（Push Notifications、Associated Domains、如启用 Sign in with Apple）。
- [ ] App Icons / LaunchScreen / Splash 资源（`npm run generate:icons:ios`）。
- [ ] 校验 App-Bound Domains（`limitsNavigationsToAppBoundDomains: true`）与 ATS（`allowMixedContent:false`）。
- [ ] `npx cap sync ios` → `npm run build:ios`（Archive）→ Transporter / Organizer 上传。

### Android 重建
- [ ] `npx cap add android` 重新生成 Gradle 工程。
- [ ] 回贴 `android/app/google-services.json` 与 `android-permissions-template`。
- [ ] 配置 `applicationId=app.summitlink`、`versionCode`/`versionName`、`signingConfigs`（对应 CI Secrets）。
- [ ] 图标 / Splash / FileProvider（如上传场景）。
- [ ] 校验 `usesCleartextTraffic=false`、`targetSdk` 满足 Google Play 当前最低要求。
- [ ] `npx cap sync android` → `npm run build:android`（`bundleRelease` 产出 AAB）。

### 本地开发环境
- Xcode 15+（仅 macOS）+ CocoaPods；Android Studio + JDK 17 + Android SDK/Build-Tools；Capacitor CLI（已固定 8.3.4）。

### CI/CD
- 复用 `.github/workflows/build-ios.yml` / `build-android.yml` / `fastlane-beta.yml`（触发：`v*` tag 或手动）。
- 所有 workflow 保持顶层 `env: FORCE_JAVASCRIPT_ACTIONS_TO_NODE24: true`（仓库约定）。
- 前提：原生工程重建后入库；签名 Secrets 配置到位（见 P1）。
