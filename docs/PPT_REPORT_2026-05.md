## Slide 1｜项目概述

### 页面标题
**SummitLink：面向户外探险的全栈平台**

### 页面可放文字
- 面向户外探险场景
- 连接用户、向导、俱乐部
- 覆盖 Web + App + 管理后台

### 讲稿
大家好，今天汇报的是我们的项目 **SummitLink**。  
它是一个面向户外探险场景的全栈平台，核心目标是连接 **普通用户、专业向导和俱乐部**。  
项目不是单点页面演示，而是一个同时具备业务流程、后台管理、支付能力和上线准备的产品雏形。

---

## Slide 2｜项目目标与业务场景

### 页面标题
**项目目标：把分散的户外服务做成完整闭环**

### 页面可放文字
- 用户：找远征、找向导、下单支付、紧急求助
- 向导 / 俱乐部：入驻、发布服务、管理订单
- 平台管理员：审核、统计、风控
- 关键词：信息整合、服务撮合、安全保障、商业闭环

### 讲稿
这个项目主要解决的是户外探险服务信息分散、流程割裂、保障不足的问题。  
对用户来说，可以完成远征浏览、向导选择、下单支付、位置追踪和 SOS 求助。  
对向导和俱乐部来说，可以进行入驻、发布服务、管理订单和处理收益。  
对平台来说，则能完成审核、统计和风险控制。  
所以项目的目标，是把探险场景中的多角色需求整合成一个真正可运行的业务闭环。

---

## Slide 3｜技术架构

### 页面标题
**技术架构：前后端分层 + 多环境部署**

### 页面可放文字
- 前端：Alpine.js + Tailwind CSS + Capacitor
- 后端：Node.js + Express + Prisma
- 数据库：开发 SQLite，生产 PostgreSQL
- 部署：腾讯云 + Railway
- CI/CD：GitHub Actions

### 讲稿
在技术选型上，我们采用的是比较轻量但完整的全栈架构。  
前端使用 **Alpine.js + Tailwind CSS**，同时结合 **Capacitor** 支持移动端 App 形态。  
后端采用 **Node.js + Express + Prisma**，数据库在开发环境使用 SQLite，在生产环境面向 PostgreSQL。  
部署层面同时支持 **Railway 海外部署** 和 **腾讯云国内节点部署**，CI/CD 通过 **GitHub Actions** 自动化完成。  
这也说明项目不只是功能开发，还考虑了上线和交付能力。

## Slide 6｜当前状态：亮点与风险

### 页面标题
**当前状态：亮点明显，风险也比较清楚**

### 页面可放文字
- 亮点
  - 功能覆盖完整
  - 三端协同清晰
  - 文档、部署、测试、CI/CD 较完整
- 风险
  - 国内支付链路仍在收口，部分场景保留 mock / fallback
  - 生产环境依赖较多，环境变量与部署配置要严格校验
  - `PII_ENCRYPTION_KEY` 迁移要谨慎

### 讲稿
当前项目最大的亮点，是功能覆盖比较完整，用户端、向导/俱乐部端和管理端都已经有了清晰能力。  
第二个亮点是工程化程度较高，文档、部署、测试和 CI/CD 基本都比较齐全。  
但风险也很明确。  
第一，**国内支付和部分商业支付场景仍然在最后收口阶段**，有些路径还保留 mock 或 fallback。  
第二，生产环境依赖比较多，上线前对环境变量和部署配置必须严格验收。  
第三，`PII_ENCRYPTION_KEY` 这类加密配置涉及历史数据迁移，处理时必须谨慎。  
也就是说，项目现在已经有很强的上线雏形，但最后一公里仍然是支付、配置和数据安全。
