### BUG-001 商业攀登筛选器点击无响应（P0）

**问题描述**  
"探索"页 → 商业攀登分类下，"全部地区" / "全部难度" / "按海拔" 三个下拉筛选器点击后无任何反应，选项从未出现。

**根本原因**  
筛选器的下拉面板（`position: absolute`）被两个祖先元素同时裁切：
1. 父行容器 `div.flex.overflow-x-auto`：CSS 规范规定当 `overflow-x` 设为 `auto/scroll` 时，`overflow-y` 计算值被强制变为 `auto`，从而在垂直方向也发生裁切；
2. 最外层 `.app-container` 带有 `overflow: hidden`，进一步阻止了超出边界的绝对定位子元素渲染。

**修复方案**  
将三个下拉面板改为 `position: fixed` + Alpine.js `$el.getBoundingClientRect()` 动态定位：
```js
const r = $el.getBoundingClientRect();
rTop = r.bottom + 4;
rLeft = r.left;
```
`position: fixed` 脱离所有 overflow 裁切上下文，始终相对于视口定位，z-index 设为 500 以覆盖所有普通内容。同时移除父行的 `overflow-x-auto`，改为 `flex-wrap`。

**复现步骤**  
1. 进入"探索"页，切换分类到"商业攀登"；
2. 点击"全部地区"按钮；
3. 修复前：无下拉出现；修复后：下拉面板在按钮正下方弹出，可正常选择。

### BUG-002 移动端天气 / 其他 API 请求失败（P0）

**问题描述**  
在 Capacitor（iOS/Android）原生壳中，天气搜索、山峰天气、营地天气等功能全部无响应，且部分订单、装备、文章接口也不可用。

**根本原因**  
上述接口的 `fetch` 调用使用了相对路径（如 `` `/api/weather?...` ``）。相对路径在 Web 浏览器中解析为 `https://your-domain.com/api/weather`，但在 Capacitor WebView 中解析为 `capacitor://localhost/api/weather`，不指向任何真实服务器，导致请求直接失败。

**修复方案**  
将所有相对路径替换为以 `API_BASE` 为前缀的绝对路径：
```js
// 修复前
fetch(`/api/weather?location=${loc}`)
// 修复后
fetch(`${API_BASE}/api/weather?location=${loc}`)
```
涉及文件：`www/index.html`，使用 `sed` 批量处理，同时手动修复了 `loadPeakWeather`（动态 URL 拼接）、`loadGear`、`loadArticles`。

**复现步骤**  
1. 在 Capacitor 打包的 App 中打开"天气"功能搜索任意地名；
2. 修复前：请求直接失败，无任何天气数据；
3. 修复后：请求打到 Railway 服务器，正常返回天气数据。

### BUG-003 登录错误信息不区分错误类型（P0）

**问题描述**  
无论是网络不通、404、401 密码错误还是 500 服务器错误，登录失败时均统一显示"网络错误，请重试"，用户无法判断问题所在。同时登录请求无超时控制，在弱网下可能无限等待。

**修复方案**  
重写 `doLogin()` 方法：
- 使用 `AbortController` 添加 15 秒超时；
- 按 HTTP 状态码区分错误：
  - `401` → "手机号或密码错误"
  - `404` → "接口不存在，请联系开发者"
  - `5xx` → "服务器错误(状态码)，请稍后重试"
  - 超时 → "请求超时，请检查网络后重试"
  - 网络断开 → "网络连接失败，请检查网络设置"
- 后端的错误 JSON `{ error: '...' }` 仍优先展示。

**复现步骤**  
1. 断开设备网络，尝试登录；修复后显示"网络连接失败，请检查网络设置"；
2. 输入错误密码登录；修复后显示"手机号或密码错误，请重试"；
3. 使用有效凭证在弱网下登录，超 15 秒显示"请求超时"。

### BUG-004 跟随系统主题切换后 UI 样式错乱（P0）

**问题描述**  
用户切换到浅色模式（或系统为浅色时自动跟随）后，大量 UI 元素仍使用硬编码的深色背景和浅色文字，导致显示效果极差（白底白字、深色卡片在浅色背景上突兀）。

**根本原因**  
原始 `body.light-mode` CSS 仅覆盖 7 条规则，遗漏了以下场景：
- `bg-slate-800/80`、`bg-slate-800/60`（带透明度的背景）
- `text-slate-100`、`text-slate-200`（几乎与白色相同）
- `border-slate-700`、`border-slate-600`（边框）
- `bottom-sheet`（底部弹窗背景）
- 底部导航栏 `nav.fixed`
- 输入框 `input`、`textarea`、`select`
- hover 状态 `.hover:bg-white/10`

**修复方案**  
重写整个浅色模式 CSS 规则集，新增约 30 条规则，全面覆盖上述场景，同时为商业攀登筛选下拉面板单独添加 `.commercial-filter-dropdown` 浅色变量。

**复现步骤**  
1. 进入"我"页面 → 点击右上角☀️/🌙切换到浅色模式；
2. 浏览首页、探索页、聊天页、个人中心各主要区域；
3. 修复后：所有区域均呈现浅色背景 + 深色文字，对比度满足可读性要求。

### BUG-005 报名/预约流程无紧急联系人管理入口（P1）

**问题描述**  
用户在预约俱乐部活动或向导服务时，必填"紧急联系人"信息，但表单内没有任何入口导航到联系人管理页；如已有保存联系人，也无法一键回填。

**修复方案**  
在 `openActivityEnroll()` 和 `openGuideServiceEnroll()` 函数中调用 `loadEmergencyContacts()`，报名 Modal 打开时自动加载已保存联系人。在表单顶部增加：
1. **快速填入 Banner**：若已有保存联系人，显示可点击的联系人名称按钮，点击即自动填入姓名和电话；
2. **"管理联系人 →"链接**：点击关闭 Modal 并跳转到"我"页面 → 紧急联系人管理区。

**复现步骤**  
1. 在"个人中心 → 紧急联系人"中先保存 1 个联系人；
2. 进入任意俱乐部页面 → 点击"报名"；
3. 修复后：报名 Modal 顶部出现蓝色"快速填入"横条，点击联系人名字即自动填入。
